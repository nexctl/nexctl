# NexCtl Architecture

## Scope

NexCtl focuses on fleet operations, remote control, auditability, and short-term status visibility. It does not include long-term metrics storage in the first phase.

## Phase 1

- Server bootstrap with MySQL and Redis connectivity
- Login API skeleton
- Node registration API and node listing API
- Current node state domain model
- WebSocket protocol baseline for agent sessions
- Agent registration and heartbeat loop
- Supervisor bootstrap scaffold
- Web login page and node list page skeleton
